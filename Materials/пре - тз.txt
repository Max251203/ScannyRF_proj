Техническое задание для разработки сайта, аналогичного СкиньСкан.рф, должно содержать подробное описание функционала, требований к интерфейсу и безопасности, а также структуру проекта и критерии реализации. Ниже представлен пример структурированного техзадания для такого сервиса.

## Общая информация

- Название проекта: Сервис быстрой вставки подписи и печати в электронные документы
- Цель: Обеспечить пользователям возможность подписывать PDF, JPG, PNG, DOC документы онлайн с помощью загрузки сканов/фото подписи и печати без необходимости установки ПО.[7][15]

## Функциональные требования

- Загрузка документов (PDF, JPG, PNG, DOC).
- Загрузка изображения подписи и/или печати (JPG, PNG, фото/скан).
- Автоматическое удаление фона с подписи и печати, корректное наложение на документ.
- Возможность перетаскивания и масштабирования подписи/печати в документе.
- Просмотр результата и его редактирование до сохранения.
- Скачивание итогового документа: с водяным знаком (бесплатно) или без водяного знака по подписке/оплате.
- Автоматическое удаление загруженных файлов через определённый срок (например, 24 часа) для безопасности.
- Авторизация: через email, соцсети (Google, VK).
- Оповещение (email, push-уведомления) о завершении обработки.
- Мобильная версия сайта/adaptive UI.

## Технические требования

- Сервер с поддержкой HTTPS/SSL; безопасность хранения и передачи данных.
- Совместимость с современными браузерами (Chrome, Firefox, Safari, Edge).
- Скорость отклика интерфейса – не более 2 секунд для базовых операций.
- Защита от SQL-инъекций, XSS, CSRF.
- Масштабируемая архитектура, подсистема хранения для краткосрочного хранения файлов.
- Реализация OCR для распознавания документов (по необходимости).
- Логирование операций для аудита.

## Требования к дизайну

- Чистый, интуитивно понятный интерфейс, максимально упрощающий рабочий процесс.
- Карточный стиль для документов, минимализм, корпоративные цвета.
- Анимации перетаскивания элементов, всплывающие окна для редактирования.
- Поддержка мобильных экранов, touch взаимодействия.

## Безопасность и конфиденциальность

- Файлы автоматически удаляются после обработки (максимум — 24 часа).
- Документы не сохраняются для последующего доступа; никаких резервных копий.
- Вся обработка изображений и документов — только на сервере, без передачи третьим лицам.

## Структура сайта

- Главная страница: описание сервиса, инструкция, кнопка загрузки документа.
- Личный кабинет: история операций, настройки профиля.
- Страница обработки: редактор наложения подписи/печати.
- Страница оплаты/доступа к подписке.
- Справочная секция/FAQ.
- Контакты и поддержка.
